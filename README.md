## 使用

下载对应的可执行文件

> [!TIP]
> **不支持MacOS、过度精简和非常冷门的特殊系统**

### 单模组快速下载

```bash
mcmdr [模组名称] [MC版本] [模组加载器] [保存目录]
```

### 多模组批量下载

```bash
mcmdr -f [批量文件 (.mcmds)]
```

#### 批量文件（.mcmds）格式

文件后缀：.mcmds
文件内容：
```mcmds
[MC版本]
[模组加载器]
[保存目录]
---
[一行一个模组，以此类推]
```

> [!NOTE]
> - 模组源：Modrinth（在国内速度不错）
> - 如果无法下载/下载模组不对，请尝试换个名称
> - 下载的模组始终是最新的
> - 这个文件格式可以用在你的MC启动器上
> - 根目录的`test.mcmds`是一个很好的例子

### 高级批量

```bash
mcmdr -f [批量文件 (.mcmds)] -c [并发数（默认4）]
```

> [!WARNING]
> 并发数**不要超出10**，在批量模式下，过高的并发数和过多的模组数可能触发平台**风控**，进而导致部分模组无法正常下载

### 实例

```bash
# 批量下载
mcmdr -f test.mcmds -c 5
```

## 常见问题

- 无适用版本……
> 这个模组还没有为你设置的MC版本发布**可用版本**，可以等几个星期

- 无稳定版……
> 这个模组还没有为你设置的MC版本发布**稳定版本**，可以等几个星期

- 无文件……
> 这个模组还没有为你设置的MC版本发布文件，可以等几个星期

- 无下载链接……
> 这个模组还没有为你设置的MC版本发布下载链接，可以等几个星期

- 任务异常……
> 可以试试重新运行下载，或者把输出内容一起复制过来提个issue

- 获取版本列表失败……
> 大概率是网络问题，可以试试挂个加速器
> 
> 如果实在不行，把输出内容一起复制过来提个issue

- 搜索……失败……
> 大概率可能是网络问题，可以试试挂个加速器
> 
> 如果实在不行，把输出内容一起复制过来提个issue

- 未找到……
> 一般是模组名输错了

- 文件不存在……
> 批量文件路径可能有误

- 格式错误……
> 批量文件格式有误，可以回到[这里](#批量文件mcmds格式)看看格式

**剩下的一般是简单的使用方法问题，这里不做说明**

## 不常见问题

**小概率出现的、一错报一大堆的（通常是英文）**归为**不常见问题**

待输出完成后，把输出**全部**复制下来，提出issue，issue还要包括：

- 操作系统及版本
- （如果有）使用的mcmds文件
- （如果有）复现说明
